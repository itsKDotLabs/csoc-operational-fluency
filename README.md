# CSOC Operational Fluency – 30 Day Lab Series

## Overview

This repository documents a structured 30-day hands-on progression focused on strengthening operational fluency in Cloud Security Operations (CSOC).

The goal is to deepen practical investigative capability across:
- Linux process analysis
- Network artifact correlation
- Cloud telemetry interpretation
- Detection engineering fundamentals
- Adversary simulation & response

Each day builds incrementally on prior skills, reinforcing repeatable investigative workflows through deliberate execution and documentation.

---

## Environment

- Two-box EC2 lab (attacker + target)
- Ubuntu
- AWS SSM for controlled access
- auditd enabled for execution telemetry
- VPC internal networking
- GitHub for structured documentation discipline

---

## Structure

Each day includes:
- Attack simulation (where applicable)
- Investigation workflow
- Artifact analysis
- Detection insight
- Structured documentation

Example progression:
- Day 01 – Environment baseline & validation
- Day 02 – Reverse shell simulation & artifact correlation
- Day 03 – (next module)

---

## Why This Project Exists

This repository reflects a deliberate effort to sharpen operational fluency in cloud security environments.

Rather than focusing solely on tools execution, the emphasis is on:
- Process and network artifact correlation
- Telemetry-driven investigation
- Behavioral detection logic
- Confidence-based analytical reasoning

The goal is consistent, documented repetition that real-world investigative workflows across host and cloud layers.
